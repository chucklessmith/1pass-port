# Background

Recently, we've seen the release of MacOS Tahoe and the near-simultaneous announcement of 1Password raising annual prices. At our house, we figured it was time to at least give the Apple Keychain tech a spin. It allows for some baseline, critical functionality like a shared repository, passkey support, etc. 

I've found it also lacks a good chunk of the feature set of 1Password, like custom fields, nonce security question responses, and various other mechanisms which can mostly be replaced with other chunks of cloud infrasturcture.

# Purpose

The CSV export functionality from 1Password and the import functionality in Apple Passwords together leave a lot of important information behind. It doesn't really do a good job of *warning* you that you're missing key data, and it sure doesn't make it *easy* to move that information into the new platform (this applies, I assume, kind of across the board, not just Apple's toolset).

This is some code I threw together to extract this information from the 1pux (full) data export generated by the 1Password client.

Currently it looks for the security questions and corresponding answers you've presumably made up to store as Security Questions fields in 1Password. It will also find and extract your recovery codes if you've stored them in the Recovery Code field.

# Use

`python main.py [YOUR_1PUX_FILE]`

There currently are no command line options. This script will unzip the file you give it, comb through the JSON data, and spit the results back at you via stdout.
